## 2. 正态分布初始化

```python
self.W = rng.normal(0, 0.1, size=(in_features, out_features))
```

### 什么是正态分布

也叫高斯分布，呈钟形曲线：

```
         ▲
         │     ***
         │    *   *
         │   *     *
         │  *       *
         │ *         *
         └─────────────▶
```

大部分数据集中在均值附近，越远离均值数据越少。

### 参数含义

| 参数 | 值 | 含义 |
|---|---|---|
| 均值 (μ) | `0` | 钟形曲线的中心，数据围绕 0 聚集 |
| 标准差 (σ) | `0.1` | 曲线的"胖瘦"，约 68% 的数据在 ±0.1 以内 |

### 为什么用正态分布初始化权重

| 目的 | 原因 |
|---|---|
| 打破对称性 | 全 0 初始化会让所有神经元学一样的东西 |
| 值要小 | 太大导致激活值饱和、梯度爆炸 |
| 均值接近 0 | 避免输入整体偏移 |

## 3. in_features 和 out_features

```python
def __init__(self, in_features: int, out_features: int, seed: int = 0):
```

### 权重矩阵的形状

```
W.shape = (in_features, out_features)

          out_features (列数)
        ┌─────────────┐
        │             │
in_     │     W       │
features│             │
(行数)  │             │
        └─────────────┘
```

### 不同值的影响

| in | out | 效果 | 参数量 |
|---|---|---|---|
| 10 | 5 | **降维**（10→5） | 50 |
| 2 | 10 | **升维**（2→10） | 20 |
| 5 | 5 | **同维变换** | 25 |

### 数据变换

```
X: (N, in) @ W: (in, out) → Y: (N, out)
```

- `in_features` = 输入有几个特征
- `out_features` = 输出有几个特征
- 两者不需要相等，W 自动完成维度转换

## 4. 形状（Shape）是什么

形状 = 张量在每个维度上的长度，用一个元组表示：

```python
np.array([1, 2, 3]).shape           # (3,)    → 一维：一条线
np.array([[1,2],[3,4],[5,6]]).shape # (3, 2)  → 二维：一个表格
```

### 代码中的形状约定

```python
X: (N, in_features)       # N 条样本，每条 in_features 个特征
W: (in_features, out_features)   # 权重矩阵
B: (out_features,)                 # 偏置向量（一维）
Y: (N, out_features)      # N 条样本，每条 out_features 个输出
```

**为什么关注形状？** 神经网络中 80% 的 bug 就是形状不匹配。

---

## 5. 张量运算的形状匹配规则

### 矩阵乘法 `@`

**规则：左张量的最后一维 = 右张量的倒数第二维（中间数要一样）**

```
(A, B) @ (B, C) = (A, C)
      ↑    ↑
    必须相等
```

```python
(4, 3) @ (3, 2)  →  (4, 2)   ✅  3 = 3
(4, 3) @ (2, 3)  →  报错      ❌  3 ≠ 2
```

### 加法 `+`

**规则：从右往左逐维比较，每维必须相等或其中一个是 1（广播）**

```python
(4, 3) + (4, 3)   ✅  完全相同
(4, 3) + (3,)     ✅  3=3，一维自动广播
(4, 3) + (4, 1)   ✅  4=4, 3 vs 1 → ok
(4, 3) + (2, 3)   ❌  4 ≠ 2
```

### 转置 `.T`

行列交换：

```python
W.shape      # (3, 2)
W.T.shape    # (2, 3)
```

## 6. 张量的维度

| 维度 | 形状 | 场景 | 类比 |
|---|---|---|---|
| **0 维** | `()` | 一个数字 | 标量 |
| **1 维** | `(5,)` | 偏置向量 | 一条线 |
| **2 维** | `(4, 3)` | 表格数据 | 一张表 |
| **3 维** | `(32, 100, 64)` | 序列数据（NLP） | 一叠表 |
| **4 维** | `(32, 3, 224, 224)` | 图像（CNN） | 一批图片 |
| **5 维** | `(8, 3, 24, 224, 224)` | 视频 | 一批视频 |

### 高维 @ 运算

numpy 的 `@` **只看最后两维**做矩阵乘法，前面的当"批次"处理：

```python
A.shape = (10, 4, 3)    # 10 个 (4×3) 的矩阵
B.shape = (10, 3, 2)    # 10 个 (3×2) 的矩阵
C = A @ B               # → (10, 4, 2)
```

等价于对每个批次单独做矩阵乘法。

## 7. 参数数量

**参数数量 = 形状里所有数字相乘**

| 维度 | 形状 | 参数数量 |
|---|---|---|
| 1 维 | `(5,)` | 5 |
| 2 维 | `(3, 2)` | 3 × 2 = 6 |
| 3 维 | `(10, 4, 3)` | 10 × 4 × 3 = 120 |
| 4 维 | `(32, 3, 224, 224)` | 32 × 3 × 224 × 224 = 4,816,896 |

### 大模型中的 "4B"

**B = Billion = 十亿**，4B = 40 亿个参数。

一个 Linear 层的参数量 = `in × out + out`，大模型是很多 Linear 层叠起来：

```
Layer 1: (4096, 4096) →  ~17M 参数
Layer 2: (4096, 4096) →  ~17M 参数
...
32 层加起来 ≈ 40 亿 = 4B
```

参数越多越聪明，但也越慢、越占显存。

## 9. 偏置初始化

```python
self.B = np.zeros(out_features)
```

### 为什么初始化为 0？

| 参数 | 初始化 | 原因 |
|---|---|---|
| **W（权重）** | 小随机 | 打破对称性 |
| **B（偏置）** | 全 0 | 不需要打破对称，从 0 开始学 |

### B 的长度

```
B 的长度 = W 的列数 = Y 的列数 = out_features
```

**W 的列数是"总设计师"**，它决定了 Y 的列数和 B 的长度。

### 广播相加

```python
X @ W  →  (N, out)    ← 二维
+ B    →  (out,)      ← 一维，自动广播到 (N, out)
```

```
(N, out)         (out,)          (N, out)
┌────────┐       ┌─────┐       ┌────────┐
│y1  y2 │        │b1  b2│       │y1+b1  y2+b2│
│y1  y2 │   +           →   │y1+b1  y2+b2│
│y1  y2 │        (广播复制)     │y1+b1  y2+b2│
│y1  y2 │                     │y1+b1  y2+b2│
└────────┘                     └────────┘
```

### 数学含义

```
y = wx + b   ← 初中一次函数
     ↑   ↑
    斜率  截距

W 控制关系和权重（斜率），B 控制整体偏移（截距）
```

## 10. K、Q、V 是什么

**K、Q、V 就是三个独立的 Linear 层的输出**，名字来自数据库检索的类比：

| 字母 | 英文 | 含义 | 类比 |
|---|---|---|---|
| **Q** | Query | 查询 | 搜索框输入的词 |
| **K** | Key | 键 | 数据库记录的标签 |
| **V** | Value | 值 | 标签对应的实际内容 |

```
              ┌─ Linear_q (W_q, B_q) ─→ Q
输入 X ───────┼─ Linear_k (W_k, B_k) ─→ K
              └─ Linear_v (W_v, B_v) ─→ V
```

### 为什么是三个？

因为搜索最少需要三个角色：
1. **用什么搜？** → Query
2. **和什么比？** → Key
3. **搜到后返回什么？** → Value

### 数学上做的事情

```python
Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d)) @ V
                     ↑                     ↑
                  算相关度权重            加权求和 V
```

- **Q·K**：算 token 之间的相关性（向量点积）
- **softmax**：把相关性变成权重（概率分布）
- **权重·V**：按权重聚合信息

**Q/K 决定"谁看谁"，V 决定"看的时候要什么内容"。**

## 11. forward 方法

```python
def forward(self, X: np.ndarray) -> np.ndarray:
    self._cache_X = X
    return X @ self.W + self.B
```

**前向传播 = 用当前权重把输入算成输出**，也就是神经网络"猜答案"的过程。

```python
self._cache_X = X   # 缓存 X，给 backward 用
return X @ self.W + self.B   # 计算 Y
```

## 12. 梯度（Gradient）

**梯度 = 告诉每个参数"往哪个方向调，能让 loss 变小"。**

### 一个参数的情况

```
loss
  ▲
  │    *  ← w=2.0，loss=5.0
  │   /
  │  /
  │ /
  │/
  └────────▶ w
```

```
梯度 > 0 → w 往小了走，loss 变小
梯度 < 0 → w 往大了走，loss 变小
梯度 = 0 → 到最低点了
```

### 多个参数

```python
dW = [[0.1, -0.2],   # 每个位置告诉你该往哪调
      [0.3,  0.0],
      [-0.1, 0.5]]
```

**每个参数有独立的梯度，正负号=方向，大小=该调多少。**

### 更新公式

```python
W = W - learning_rate * dW
```

就像**蒙眼下山**：脚往各个方向踩一踩，找出下坡方向，走一步。

## 13. dY 是怎么来的

```python
# main() 中:
pred = layer.forward(X)           # 先算预测
loss, dY = mse_loss(pred, target) # ← dY 从这里来
layer.backward(dY)                # 再传给 backward
```

```python
def mse_loss(pred, target):
    diff = pred - target
    loss = np.mean(diff**2)
    grad = 2 * diff / diff.size   # ← 这就是 dY
    return loss, grad
```

**dY = loss 对预测值 Y 的导数**，也叫"上游梯度"，告诉 backward "Y 该往哪调才能让 loss 变小"。

**梯度完全依赖 loss**，换不同的 loss 函数，梯度就不一样。

---

## 14. backward 方法

```python
def backward(self, dY: np.ndarray):
    X = self._cache_X
    self.dW = X.T @ dY          # W 的梯度
    self.dB = dY.sum(axis=0)    # B 的梯度
    dX = dY @ self.W.T          # X 的梯度
    return dX
```

### 为什么用 `X.T`？

形状匹配需要转置：

```
X:      (4, 3)   直接 @ dY: (4, 2) ❌ 3 ≠ 4
X.T:    (3, 4)   @ dY: (4, 2) ✅ 4=4，结果 (3,2) = W.shape
```

### 为什么返回 dX？

`dX` 是**传给上一层的梯度接力棒**：

```
输入 → Layer1 → Layer2 → Layer3 → loss
                        ↑
              Layer3.backward 返回 dX
                        ↓
              Layer2.backward(dX) 继续往前传
```

**每一层的 backward 返回 dX，让梯度能一路反传到输入。**

## 15. 什么时候更新 W、B？

你的代码里**没有写更新步骤**，它只做：前向 + 反向 + 梯度校验。

### 完整训练循环

```python
layer = Linear(in_f, out_f)

for epoch in range(100):
    pred = layer.forward(X)              # 1. 前向：猜答案
    loss, dY = mse_loss(pred, target)    # 2. 算 loss
    layer.backward(dY)                   # 3. 反向：算梯度
    layer.update(lr=0.01)                # 4. 更新参数 ← 缺的这一步
```

```python
def update(self, lr=0.01):
    self.W -= lr * self.dW    # 每个元素各自减
    self.B -= lr * self.dB
```

## 16. 如何验证反向传播对不对

### 数值梯度对比

```python
# 两种方法算梯度，结果一样 = 对
num_dW = numerical_gradient(loss_only, layer.W)  # 笨办法：动一下看变化
err_W = np.max(np.abs(num_dW - layer.dW))        # 比较差异
assert err_W < 1e-5   # 误差 < 十万分之一 = 通过
```

### 为什么能验证？

| 方法 | 原理 | 可信度 |
|---|---|---|
| **数值梯度** | 每个参数 +ε/-ε，看 loss 变化 | ★★★★★ 定义本身 |
| **反向传播** | 链式法则手推公式 | ★★★ 可能推错 |

如果笨办法和聪明办法算出来一样 → 聪明办法一定对。

### 误差阈值

| 误差 | 结论 |
|---|---|
| **≤ 1e-5** | ✅ 对，放心用 |
| **1e-4** | ⚠️ 大概率有 bug |
| **≥ 1e-3** | ❌ 肯定有 bug |

## 速查表

```
┌─────────────────────────────────────────────────┐
│  X: (N, in)    输入：N 条样本，in 个特征          │
│  W: (in, out)  权重：行=输入，列=输出             │
│  B: (out,)     偏置：长度=输出特征数              │
│  Y: (N, out)   输出：N 条样本，out 个特征         │
├─────────────────────────────────────────────────┤
│  正向: Y = X @ W + B                            │
│  反向: dW = X.T @ dY                            │
│        dB = sum(dY)                             │
│        dX = dY @ W.T                            │
│  更新: W -= lr * dW                             │
│        B -= lr * dB                             │
├─────────────────────────────────────────────────┤
│  矩阵乘法: (A,B) @ (B,C) = (A,C)                │
│  广播规则: 从右往左，每维相等或为 1               │
│  参数数量: 形状里所有数字相乘                     │
│  梯度校验: 数值 vs 解析，误差 < 1e-5             │
└─────────────────────────────────────────────────┘
```
