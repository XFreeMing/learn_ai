# rng.normal 与正态分布初始化（学习心得）

> 对应代码：`01_handwritten_nn/01_linear_y_wx_b.py` 第 23 行
> `self.W = rng.normal(0, 0.1, size=(in_features, out_features))`
> 可交互可视化：`01_handwritten_nn/rng_normal_interactive.html`

## 1. `rng.normal(loc, scale, size)` 三个参数

| 位置 | 值 | 名称 | 作用 |
|------|------|------|------|
| 第 1 个 | `0` | `loc`（均值 μ） | 分布**中心位置**，采样值围绕它对称分布 |
| 第 2 个 | `0.1` | `scale`（标准差 σ） | 分布**胖瘦/分散度**，越大越分散；必须 ≥ 0 |
| 第 3 个 | `size=(in, out)` | 输出形状 | 生成 `in × out` 的**矩阵**，每个元素独立采样 |

正态分布概率密度：

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}}\, e^{-\frac{(x-\mu)^2}{2\sigma^2}}$$

经验法则：约 **68%** 落在 `[μ−σ, μ+σ]`，**95%** 落在 `[μ−2σ, μ+2σ]`。

## 2. 不同取值如何影响输出

- 改 `loc` → 整条钟形曲线左右**平移**（中心移动）。
- 改 `scale` → 曲线变胖/变瘦：
  - `scale=0.1` → 权重多在 `[-0.2, 0.2]`，初始化温和。
  - `scale=1.0` → 权重多在 `[-2, 2]`，幅度大 10 倍，易梯度爆炸/震荡。
- 改 `size` → 只决定生成多少个数、什么形状，**不改变每个数的统计规律**。
- N 越大 → 实测均值/标准差越接近设定的 `loc`/`scale`（大数定律）。

**为什么初始化用小 σ（0.1）？** 权重太大 → 激活值过大 → 落入激活函数饱和区（梯度≈0）→ 学不动。小 σ「避免一开始就饱和」。

## 3. 输出是矩阵；直方图是它的「分布统计」

- 输出确实是一个**矩阵** `W`，每个格子 `W[i,j]` 是一次独立采样。
- 直方图 **不是矩阵本身**，而是把矩阵里全部 N 个数字「按大小分桶计数」：
  - **横轴 = 采样值大小**（数轴切成等宽区间/桶）
  - **纵轴 = 落在该区间的元素个数**（频数 = 柱高）
- 矩阵 → 直方图四步：① 拿矩阵 → ② 数轴切成若干桶 → ③ 数每个桶里几个数 → ④ 计数即柱高。
- 一个采样值只会落进**一个**桶；理论曲线对应的期望频数 = `N × 桶宽 × pdf(x)`。

## 4. 配色为什么改 loc/scale 不变？（相对 vs 绝对）

可视化里矩阵格子颜色用了「相对配色」：`t = (v − loc) / (3σ)`。

因为 `v = loc + σ·z`，代入后 `loc`、`σ` 完全约掉，只剩 `z/3`（标准正态随机数，由 **seed** 决定）。

| 配色模式 | 公式 | 谁决定颜色 |
|----------|------|-----------|
| 相对(z分数) | `t = (v−loc)/(3σ)` | loc/scale 被约掉，**只受 seed 影响** |
| 绝对(固定数轴±2) | `t = v/2` | **直接看数值**，改 loc/scale 颜色跟着变 |

结论：相对配色看的是「偏离自己均值几个 σ」，所以平移/缩放分布颜色不变；只有换 seed（重新采样）才换颜色。绝对配色才会随数值大小变化。
